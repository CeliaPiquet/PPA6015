Github

Autonomie

Qu'est-ce que Github ?

Github est un service web qui permet d'heberger du code et d'offrir un dépôt distant Git. Il est directement accessible sur https://github.com/

Github est LA plateforme de contrôle de versions la plus grosse et populaire à l'heure actuelle. Si vous n'avez jamais utilisé Git, commencez par cette plateforme car elle est aussi la plus documentée et donc la plus facile d'accès.

On recommande Github surtout pour les projets personnels, les projets de petites et moyennes entreprises et les projets open-source.

Avantages et limites du service

Comment utiliser Github ?

  1. Allez sur le site https://github.com/
  2. Cliquez sur Sign-up pour accéder à la page d'inscription et remplissez le formulaire.
  3. Une fois le formulaire rempli et envoyer, vous allez recevoir un code de confirmation par mail qu'il va falloir entrer dans la page suivante.
  4. Vous pouvez Skip personalization cela n'a pas d'intérêt.
  5. Vous avez maintenant votre tableau de bord face à vous.
  1. Une fois connecté, cliquez sur + puis sur New repository en haut à droite de la fanêtre.
  2. Remplissez le formulaire en nommant votre dépôt et en laissant la visibilité en public, cela me permettre d'y accéder facilement.
    Cliquez ensuite sur Create repository.
  3. Bravo votre dépôt distant est créé.
  1. Placez vous dans un répertoire vide qui contiendra vos fichiers de code puis ouvrez, depuis cet endroit, Git Bash (ou un Terminal le cas échéant).
  2. Entrez la commande git init. Celle-ci va créer un fichier caché .git faisant ainsi de votre répertoire un dépôt local Git.
  3. Pour tester notre VCS et la bonne interaction de nos deux dépôts, il faut manipuler des fichiers.
    Créez donc un fichier HTML avec la commande touch index.html. Vous pouvez y écrire ce que vous voulez à l'intérieur.
  4. Entrez la commande git add index.html. Celle-ci ajoute notre fichier à notre Stage (cf. section Git pour rappeler cette étape).
  5. Entrez la commande git commit -m "Ceci est mon premier commit". Vous pouvez bien sûr mettre n'importe quel message entre les guillemets.
    Cette étape empaquette notre fichier comme une lettre mise dans une enveloppe. L'option -m permet d'inscrire des informations sur ce paquet, comme on pourrait inscrire une adresse sur notre enveloppe.
  6. Entrez la commande git branch -M main. Celle-ci permet de créer une branche de travail principale appelée main. Dans Git, une branche est une ligne temporelle sur laquelle sont sauvegardées différentes versions de code. Vous pourrez en apprendre plus sur le système de branche dans la vidéo dédiée à ce sujet dans les ressources supplémentaires.
  7. Entrez la commande git remote add origin url_du_dépôt_distant en y inscrivant l'url de votre dépôt distant (celle-ci est disponible sur la page de votre dépôt et finit par .git).
  8. Enfin, entrez la commande git push -u origin main afin d'envoyer votre nouvelle branche main sur votre dépôt distant.
  9. Si maintenant vous revenez sur votre page Github et que vous l'actualisez, vous devriez voir votre fichier index.html ainsi que le message de votre commit.
