Gitlab

Autonomie

Qu'est-ce que Gitlab ?

Gitlab est un service web qui permet d'heberger du code et d'offrir un dépôt distant Git. Il est directement accessible sur https://gitlab.com/.

Avantages et limites du service

Comment utiliser Gitlab ?

  1. Créez un compte sur https://gitlab.com/ et connectez-vous.
  2. Créez un nouveau blank projet public et copiez l'url de votre dépôt.
  3. Placez vous dans un répertoire vide qui contiendra vos fichiers de code puis ouvrez, depuis cet endroit, Git Bash (ou un Terminal le cas échéant).
  4. Entrez la commande git init. Celle-ci va créer un fichier caché .git faisant ainsi de votre répertoire un dépôt local Git.
  5. Pour tester notre VCS et la bonne interaction de nos deux dépôts, il faut manipuler des fichiers.
    Créez donc un fichier HTML avec la commande touch index.html. Vous pouvez y écrire ce que vous voulez à l'intérieur.
  6. Entrez la commande git add index.html. Celle-ci ajoute notre fichier à notre Stage (cf. section Git pour rappeler cette étape).
  7. Entrez la commande git commit -m "Ceci est mon premier commit". Vous pouvez bien sûr mettre n'importe quel message entre les guillemets.
    Cette étape empaquette notre fichier comme une lettre mise dans une enveloppe. L'option -m permet d'inscrire des informations sur ce paquet, comme on pourrait inscrire une adresse sur notre enveloppe.
  8. Entrez la commande git branch -M main. Celle-ci permet de créer une branche de travail principale appelée main. Dans Git, une branche est une ligne temporelle sur laquelle sont sauvegardées différentes versions de code. Vous pourrez en apprendre plus sur le système de branche dans la vidéo dédiée à ce sujet dans les ressources supplémentaires.
  9. Entrez la commande git remote add origin url_du_dépôt_distant en y inscrivant l'url de votre dépôt distant (celle-ci est disponible sur la page de votre dépôt et finit par .git).
  10. Enfin, entrez la commande git push -u origin main afin d'envoyer votre nouvelle branche main sur votre dépôt distant.
  11. Si maintenant vous revenez sur votre page Gitlab et que vous l'actualisez, vous devriez voir votre fichier index.html ainsi que le message de votre commit.
