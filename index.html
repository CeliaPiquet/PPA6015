Leçon n°5 : La gestion de versions


Objectifs de la leçon

  • Définir ce qu'est un VCS
  • Expliquer comment fonctionne Git
  • Utiliser Git basiquement : configuration, manipulation et gestion de conflits
  • Utiliser un service web basé sur Git : Github, Gitlab ou Bitbucket

Introduction

Pour comprendre l'importance de la gestion de versions, regardez la situation exposée dans la vidéo ci-dessus :


Extrait tiré de la vidéo originale : https://youtu.be/gp_k0UVOYMw

On s'aperçoit que plusieurs problèmes ont eu lieu avec cette façon de collaborer :

  • perte de temps entre les échanges
  • perte du travail et donc retour à 0

Il est aussi important de noter que dans cette situation il n'y a que 2 fichiers mis en jeux, alors que dans un projet réel vous manipulerez souvent entre 10 et 30 fichiers. Les problèmes alors mentionnés auraient de plus grandes conséquences dans un cas réel.


Solution

On pourrait alors penser à des applications comme Google Drive ou OneDrive pour partager en temps réel ses fichiers mais ces logiciels ne sont pas faits pour du code et ne permettent pas de garder une trace de toutes les versions d'un document. Aussi, avec un nombre conséquent de fichiers à manipuler leur utilisation devient très longue et compliquée.

On va donc utiliser dans le monde informatique ce qu'on appelle des logiciels de gestion de versions, aussi abrégés VCS pour Version Control System en anglais.

Cette leçon a donc pour but de vous faire comprendre comment fonctionnent les logiciels de gestion de versions et de vous les faire manipuler.
